# Requirements Document

## Introduction

`chroma-sync` は、ai/psdファイルをJPEGに変換する際の色ずれ問題を自動解決するアプリケーションです。本ドキュメントでは、アプリケーションの機能要件・非機能要件を定義します。

## Alignment with Product Vision

本要件は `product.md` で定義した以下の目標を達成するために定義されています：
- 色の忠実性を保ちながらJPEG変換を行う
- 変換後の色ずれを自動検出・修正する
- ユーザーが圧縮率を指定できる

---

## Requirements

### Requirement 1: ファイルアップロードと形式検証

**User Story:** デザイナーとして、ai/psdファイルをアプリにアップロードして変換を開始したい。そうすることで、手動でのファイル変換作業が不要になる。

#### Acceptance Criteria

1. WHEN ユーザーがai/psdファイルをアップロードする THEN システムはファイル形式を検証し、対応形式であることを確認する
2. IF アップロードされたファイルが対応形式でない THEN システムはエラーメッセージを表示し、対応形式（.ai, .psd）を案内する
3. WHEN 複数ファイルが選択された THEN システムはバッチ処理キューに追加し、順番に処理する

---

### Requirement 2: 圧縮率・目標サイズの指定

**User Story:** デザイナーとして、変換後のJPEGの目標ファイルサイズまたは品質レベルを指定したい。そうすることで、用途に合った最適なサイズと品質の画像を得られる。

#### Acceptance Criteria

1. WHEN ユーザーが目標ファイルサイズ（例: 500KB）を指定する THEN システムはその範囲内で最高品質のJPEGを生成する
2. WHEN ユーザーが品質レベル（1〜100）を指定する THEN システムは指定品質でJPEGを生成する
3. IF 目標サイズと品質の両方が指定された THEN システムは目標サイズを優先する
4. WHEN ユーザーが何も指定しない THEN システムはデフォルト品質（85）で変換する

---

### Requirement 3: 色味保持変換

**User Story:** デザイナーとして、ai/psdファイルをJPEGに変換した後も元の色味が保たれていてほしい。そうすることで、変換後の再調整作業が不要になる。

#### Acceptance Criteria

1. WHEN ファイルを変換する THEN システムはソースファイルのカラープロファイル（ICCプロファイル）を読み取り、sRGBに正確に変換する
2. WHEN 変換が完了する THEN システムは変換前後の色差（ΔE）を計算する
3. IF 色差が閾値（ΔE < 2.0）以内 THEN システムは変換成功と判定し、ファイルを出力する
4. IF 色差が閾値を超える THEN システムは自動色味修正処理を実行する

---

### Requirement 4: 自動色味修正

**User Story:** デザイナーとして、変換後に色ずれが発生した場合、自動的に修正してほしい。そうすることで、手動での色調整作業が不要になる。

#### Acceptance Criteria

1. WHEN 色差が閾値を超えた領域が検出される THEN システムは該当領域の色補正を自動実行する
2. WHEN 自動色味修正が完了する THEN システムは修正後の色差を再計算し、ΔE < 2.0 以内になっているか確認する
3. WHEN 最終的な変換結果が生成される THEN システムは色差レポート（修正箇所、修正前後のΔE値）を生成する

---

### Requirement 5: 変換結果の表示とダウンロード

**User Story:** デザイナーとして、変換後のJPEGをプレビューして確認し、ダウンロードしたい。そうすることで、変換結果を確認してから使用できる。

#### Acceptance Criteria

1. WHEN 変換が完了する THEN システムは変換前後の画像を並べて表示する
2. WHEN プレビューが表示される THEN システムは変換後ファイルサイズ、色差（ΔE）、適用した色補正の概要を表示する
3. WHEN ユーザーがダウンロードボタンをクリックする THEN システムはJPEGファイルをダウンロードする
4. WHEN バッチ処理が完了する THEN システムは全ファイルをZIPにまとめてダウンロードできるようにする

---

## Non-Functional Requirements

### Code Architecture and Modularity
- **Single Responsibility Principle**: 各モジュールは単一の責任を持つ（変換処理、色差計算、UI表示を分離）
- **Modular Design**: 変換エンジン、色補正モジュール、UIを独立したモジュールとして設計
- **Dependency Management**: コア変換ロジックはUIフレームワークに依存しない
- **Clear Interfaces**: 変換エンジンはAPIを通じて呼び出す設計

### Performance
- A4サイズ（300dpi）の1ファイルを30秒以内に変換完了
- バッチ処理時は並列処理で効率化
- Webアプリはファイルサイズ 100MB 以下のファイルに対応

### Security
- アップロードされたファイルはサーバー上で処理後、一定時間経過したら自動削除
- ファイルのウイルスチェック（本番環境）
- HTTPS通信必須（Web版）

### Reliability
- 変換処理のエラー時は元ファイルを保持し、エラー内容をユーザーに通知
- 変換失敗時は適切なエラーメッセージとともにリトライ可能にする

### Usability
- ドラッグ&ドロップによるファイルアップロード対応
- モバイル端末からも利用可能なレスポンシブデザイン
- 変換の進捗をリアルタイムで表示（プログレスバー）
